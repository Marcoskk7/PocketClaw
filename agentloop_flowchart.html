Nanobot AgentLoop 执行流程

nanobot agent -m "your message" | python -m nanobot agent

ENTRY CLI 入口
用户执行 nanobot agent -m "消息" 或 python -m nanobot agent
→ cli/commands.py :: agent()
加载配置、创建 MessageBus、LLMProvider、CronService
INIT AgentLoop 初始化
agent/loop.py :: AgentLoop.__init__()
初始化: ContextBuilder · SessionManager · ToolRegistry · SubagentManager · MemoryConsolidator
注册默认工具: read_file, write_file, edit_file, list_dir, exec, web_search, web_fetch, message, spawn, cron
单次消息: process_direct() / 交互: run()
STEP 1 接收消息 & 会话管理
_process_message(msg)
① 检查 slash 命令 (/new, /help, /stop, /restart)
② 获取或创建 Session → sessions.get_or_create(key)
③ 可能触发 Memory Consolidation → maybe_consolidate_by_tokens()
④ 设置工具上下文 (channel, chat_id)
STEP 2 构建上下文
agent/context.py :: build_messages()
① 构建 system prompt: 身份 + AGENTS.md/SOUL.md + Memory + Skills
② 加入历史对话 session.get_history()
③ 附加当前用户消息 + 运行时上下文 (时间、平台信息)
④ 处理 media 附件 (图片 base64 编码)
STEP 3 调用 LLM
providers/base.py :: provider.chat_with_retry(messages, tools, model)
将 messages + tool definitions 发送到 LLM API
返回 LLMResponse: content, tool_calls, finish_reason, usage, reasoning_content
判断 response.has_tool_calls ?
YES — LLM 返回了 tool_calls → 进入工具执行分支
NO — LLM 只返回 content → 最终回复, 退出循环 (break)
YES: 有 tool_calls
STEP 4 执行工具调用
① 发送 progress 提示 (思考内容 + 工具提示)
② 将 assistant 消息 (含 tool_calls) 追加到 messages
③ 遍历每个 tool_call:
    tools/registry.py :: tools.execute(name, args)
    → 参数类型转换 → 参数校验 → tool.execute(**params)
④ 将工具结果追加到 messages → context.add_tool_result()
回到 STEP 3, iteration++
循环继续直到: LLM 不再请求工具 / 达到 max_iterations / 遇到错误
NO: 无 tool_calls / 退出循环
STEP 5 保存会话 & 发送回复
① _save_turn(session, messages, skip) — 保存新消息到 session, 截断过长工具结果
② sessions.save(session) — 持久化 session
③ maybe_consolidate_by_tokens() — 再次检查是否需要记忆整合
④ 构建 OutboundMessage 发送回复给用户
OUTPUT 用户收到回复
CLI 模式: _print_agent_response() 打印到终端 (支持 Markdown 渲染)
Gateway 模式: 通过 MessageBus → Channel (Telegram/Slack/...) 发送
入口/初始化
核心流程
工具执行
判断分支
退出/输出
